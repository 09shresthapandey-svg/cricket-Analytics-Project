Cricket Analytics Project

Introduction

Cricket is one of the most widely followed sports in the world, with billions of viewers across countries such as India, England, Australia, Pakistan, and South Africa. The game is played in multiple formats including Test cricket, One Day Internationals (ODIs), and Twenty20 (T20), each demanding different skills, strategies, and performance expectations from players. Batting, bowling, and all-round performance are measured using statistics such as runs scored, strike rate, economy rate, wickets taken, and match impact.

In recent years, the availability of large-scale cricket data has transformed how the sport is analyzed. Ball-by-ball datasets allow analysts to study scoring patterns, wicket events, and team/player performance. This project uses Cricsheet T20 ball-by-ball data and applies a full data science lifecycle: data collection, cleaning, EDA, dimensionality reduction (PCA), clustering, and association rule mining.

The goal is to uncover patterns that explain performance and match dynamics. In Module 2, PCA helps visualize and reduce complexity, clustering groups similar player batting profiles, and ARM (Apriori) finds “if-then” relationships between events within overs.

Research Questions

  • How do scoring patterns vary across overs and innings in T20 matches?
  • Which teams and players contribute the most runs and wickets in the sample?
  • Can we reduce numeric match features to 2D/3D while retaining most information?
  • Can we cluster players into different batting profile groups?
  • What events frequently occur together within an over (ARM rules)?

PCA (Principal Component Analysis)

What is PCA?

Principal Component Analysis (PCA) is a dimensionality reduction method that transforms correlated numeric variables into a smaller set of uncorrelated variables called principal components. The first principal component captures the most variance (information), followed by subsequent components that capture remaining variance while staying orthogonal. PCA is useful for visualization and reducing complexity while retaining most of the information in the data.

Requirement satisfied: PCA is applied only on numeric columns. Label columns (team names, players, wicket_type) were excluded. I used StandardScaler before applying PCA.

Numeric Features Used

Clustering

Compare & Contrast: KMeans vs Hierarchical vs DBSCAN

KMeans partitions data into k clusters using centroids and minimizes within-cluster variance. Hierarchical clustering builds nested clusters and visualizes structure through a dendrogram. DBSCAN is density-based and can label outliers as noise while discovering non-spherical clusters.

Data Prep: Ball-by-ball data was aggregated by striker to create numeric player profiles. Labels (player names) were saved separately and removed before clustering. Numeric features were standardized with StandardScaler.

ARM (Association Rule Mining)

Overview

Association Rule Mining (ARM) discovers “if-then” patterns in transaction data using metrics: support (how often a rule occurs), confidence (how likely the consequent is given the antecedent), and lift (strength relative to chance). The Apriori algorithm finds frequent itemsets above a support threshold and generates rules from them.

Transaction format used: Each over was treated as one transaction (grouped by match_id, innings, and over number derived from ball). Items included: WICKET, BOUNDARY, EXTRAS, HIGH_RUN_OVER, DOT_HEAVY.

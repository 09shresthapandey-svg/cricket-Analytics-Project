Cricket Analytics Project

Introduction

This project analyzes T20 cricket ball-by-ball data from Cricsheet using a full data science workflow. It includes data cleaning, exploratory data analysis (EDA), Principal Component Analysis (PCA), clustering (KMeans, Hierarchical, DBSCAN), and Association Rule Mining (ARM).

Data Preparation & EDA

Dataset Source: Cricsheet

Cleaning Steps:
  • Standardized column names
  • Removed _info files
  • Kept key columns (match_id, innings, ball, runs_off_bat, extras, etc.)
  • Dropped missing values
  • Saved combined_30_matches.csv

PCA (Principal Component Analysis)

PCA reduces numeric features (innings, ball, runs_off_bat, extras) into fewer components while preserving variance. StandardScaler was used before PCA.

Clustering

KMeans, Hierarchical, and DBSCAN were applied to aggregated player statistics. Silhouette method was used to choose optimal k values.

Association Rule Mining (ARM)

ARM was applied using Apriori algorithm. Each over was treated as a transaction. Metrics used: Support, Confidence, Lift.

ARM Code: View ARM Script

Conclusion

This project demonstrates a complete data science pipeline on cricket data. EDA identified scoring patterns, PCA reduced dimensionality, clustering grouped players, and ARM discovered event associations within overs.

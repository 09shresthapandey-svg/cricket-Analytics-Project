Cricket Analytics Project

Introduction

This project analyzes T20 cricket ball-by-ball data from Cricsheet using a full data science workflow. It includes data cleaning, exploratory data analysis (EDA), Principal Component Analysis (PCA), clustering (KMeans, Hierarchical, DBSCAN), and Association Rule Mining (ARM).

Association Rule Mining (ARM)

Overview

Association Rule Mining (ARM) discovers “if-then” patterns in transaction data using metrics: support (how often a rule occurs), confidence (how likely the consequent is given the antecedent), and lift (strength relative to chance). The Apriori algorithm finds frequent itemsets above a support threshold and generates rules from them.

Data Prep (Transaction Data)

ARM requires unlabeled transaction data. In this project, each over is treated as one transaction (grouped by match_id, innings, and over derived from ball). Items include: WICKET, BOUNDARY, EXTRAS, HIGH_RUN_OVER, DOT_HEAVY.

Conclusion

This project demonstrates a complete data science pipeline on cricket data. EDA identified scoring patterns, PCA reduced dimensionality, clustering grouped players, and ARM discovered event associations within overs.
